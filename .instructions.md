# Peak Haunt - Copilot コーディング規則

## 1. Backend (Python/FastAPI)

- 型ヒント: 必須
- Docstring: 必須
- Linting: ruff で統一
- テスト: pytest で全モジュール対象
- ルーティング: `/api/routers/v1/` 配下にバージョン別に整理

## 2. Frontend (Next.js/TypeScript)

- TypeScript: `tsconfig.json` の strict 設定を遵守
- コンポーネント: 型安全に実装、`components/` に機能別整理

## 3. 開発フロー

- ブランチ: 機能ごとに `feature/xxx` を作成
- コミット: Conventional Commits 形式を使用
- 環境変数: `.env` で管理（`.env.example` を VCS に含める）
- DB スキーマ変更: migration ファイルとして記録
- Docker Compose: v2 使用時は、トップレベルの `version` キーは不要

## 4. コメント・ドキュメント規則

- 出力言語: 日本語で記述
- 絵文字: 使用禁止
- コメント: 見れば分かる、考えれば分かる内容のコメントは不要
- 関数・クラス: 必ずドキュメント文字列（docstring）を付ける
- Docstring: 一行程度で簡潔に記述（例：`def get_user(user_id: int): "指定されたユーザーを取得"`）
